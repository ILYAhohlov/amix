# Шаги для завершения настройки PostgreSQL

## ✅ Что уже сделано:
1. Создана схема таблицы blog_posts
2. Создан файл подключения к БД (server/db.ts)
3. API переделан для работы с PostgreSQL
4. Фронтенд обновлён для загрузки из API
5. Создан скрипт миграции постов

## 📋 Что нужно сделать СЕЙЧАС:

### Шаг 1: Получить DATABASE_URL из Render
1. Откройте вашу базу данных на Render
2. Перейдите на вкладку "Connect" или "Info"
3. Скопируйте **Internal Database URL**

### Шаг 2: Создать локальный .env файл
Создайте файл `.env` в корне проекта со следующим содержимым:

```
DATABASE_URL=вставьте_сюда_Internal_Database_URL_из_Render
TELEGRAM_BOT_TOKEN=1519701472:AAF6Z5aFZXaaw0fw9dTgYQ1U7a0AVEEDG88
TELEGRAM_CHAT_ID=-335182799
ADMIN_PASSWORD=ваш_пароль_администратора
PORT=5000
```

### Шаг 3: Создать таблицу в базе данных
Выполните команду:
```
npm run db:push
```

### Шаг 4: Мигрировать существующие посты
Выполните команду:
```
npm run db:migrate-posts
```

### Шаг 5: Запустить проект локально
```
npm run dev
```

### Шаг 6: Протестировать
1. Откройте http://localhost:5000/blog - должны загрузиться посты
2. Откройте http://localhost:5000/admin/blog - войдите и попробуйте создать пост
3. Проверьте, что пост сохраняется

### Шаг 7: Задеплоить на Render
```
git add .
git commit -m "Migrate blog CMS to PostgreSQL"
git push
```

Render автоматически задеплоит изменения!

## ⚠️ ВАЖНО:
После деплоя на Render нужно будет выполнить миграцию постов на production:
1. В Render Dashboard откройте ваш веб-сервис
2. Перейдите в Shell (консоль)
3. Выполните: `npm run db:migrate-posts`

Или можно создать посты заново через админку на production.
