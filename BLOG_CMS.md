# Blog CMS - Руководство администратора

## Доступ к админ-панели

1. Откройте `https://amix.pro/admin/blog`
2. Введите пароль администратора
3. После успешного входа вы увидите список всех постов

## Создание нового поста

1. Нажмите кнопку **"+ Новый пост"**
2. Заполните все поля:
   - **title** - Заголовок поста (обязательно)
   - **subtitle** - Подзаголовок (обязательно)
   - **slug** - URL поста (например: `my-new-post`). Должен быть уникальным!
   - **excerpt** - Краткое описание для списка постов
   - **author** - Автор (по умолчанию: AMIX International Group)
   - **imageUrl** - Путь к изображению (например: `/images/blog/my-image.jpg`)
   - **contactEmail** - Email для связи (опционально)
   - **publishDate** - Дата и время публикации
   - **keywords** - Ключевые слова через запятую (для SEO)
   - **content** - Параграфы контента (можно добавлять/удалять)

3. Нажмите **"Сохранить"**

## Редактирование поста

1. В списке постов нажмите **"Редактировать"** на нужном посте
2. Измените необходимые поля
3. Нажмите **"Сохранить"**

## Удаление поста

1. В списке постов нажмите **"Удалить"** на нужном посте
2. Подтвердите удаление

## Форматирование контента

В параграфах контента можно использовать HTML-теги:
- `<strong>текст</strong>` - жирный текст
- `<em>текст</em>` - курсив
- `<a href="url">текст</a>` - ссылка
- `<ul><li>пункт</li></ul>` - маркированный список

Пример:
```
<strong>Важно:</strong> Vietnam offers <a href="https://example.com">great opportunities</a> for investors.
```

## Настройка пароля администратора

### В Render.com:

1. Откройте ваш сервис в Render Dashboard
2. Перейдите на вкладку **Environment**
3. Нажмите **Add Environment Variable**
4. Добавьте:
   - Key: `ADMIN_PASSWORD`
   - Value: ваш_безопасный_пароль
5. Нажмите **Save Changes**
6. Сервис автоматически перезапустится

### Локально (для разработки):

1. Создайте файл `.env` в корне проекта
2. Добавьте строку: `ADMIN_PASSWORD=ваш_пароль`
3. Перезапустите сервер разработки

## Структура хранения

Все посты хранятся в файле `client/src/data/blogPosts.json`. 

**Важно**: При деплое на Render изменения в JSON-файле НЕ сохраняются между перезапусками. Для production рекомендуется использовать базу данных.

## SEO оптимизация

Каждый пост автоматически получает:
- Meta tags (title, description, keywords)
- Open Graph tags (для Facebook, LinkedIn)
- Twitter Card tags
- Schema.org разметку (BlogPosting)

Убедитесь, что заполнены:
- `title` - для заголовка страницы
- `excerpt` - для meta description
- `keywords` - для SEO
- `imageUrl` - для социальных сетей (рекомендуемый размер: 1200x630px)

## Доступ к постам

После публикации пост будет доступен по адресу:
- Список постов: `https://amix.pro/blog`
- Отдельный пост: `https://amix.pro/blog/[slug]`

## Безопасность

- Никогда не делитесь паролем администратора
- Используйте сложный пароль (минимум 12 символов)
- Регулярно меняйте пароль
- Выходите из админ-панели после работы (кнопка "Выйти")

## Поддержка

При возникновении проблем обращайтесь к разработчику или проверьте логи в Render Dashboard.
